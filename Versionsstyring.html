Versionsstyring

Hvorfor versionsstyring?

For at undgå fejl.

For at kunne gå tilbage, hvis fejl alligevel sker.

For at holde styr på flere delprojekter.

Samme elementer kan måske laves i forskellige versioner til forskellige kunder.

For at kunne samarbejde.

Hvad er versionsstyring?

Versionsstyring er så simpelt som at holde styr på forskellige versioner af et projekt.

Det kan komme i form af navngivning af filer eller mapper.

Hvis man er en del af et større projekt med flere indblandet, der arbejder på forskellige versioner af samme projekt (små ting der f.eks. tilføjes til forsiden af en webside, hvis det er det man arbejder på), så kan det være en fordel at alle navngiver ens som ”Version 0.1” eller hvis man bruger et kodesprog ”Statisk objekt” navngives f.eks. ”SO_01” og hvis der er flere ”SO_02, SO_03” osv.

Det kan komme i form af navngivning af filer eller mapper.

Git

Git er ikke GitHub.
Git er en standard ”protokol”.

Git er et versionsstyringssystem. -> En hjemmeside der hjælper med at holde styr på koder og versioner af samme kode. Så flere har mulighed for at arbejde på samme projekt, og vise hvad de har fikset og kodet på.

GitHub er en webplatform der tillader at man kan oprette Git-repositories (Skyen), altså en server, så du ikke selv behøver at installere en.

Det er tænkt som et kodedelings-site Det er et socialt medie, I det at man kan kommentere på hinandens kode og komme med forbedringer, ændringer eller spørger andre om hjælp til ens kode.

GitHub har en del features der har fordele til samarbejde og projektstyring mv. Man kan hente store og små stykker kode til egne projekter.

Hvad er GitHub?

  • Er en webplatform der tillader at man kan oprette Git-repositories.
  • Har en del flere features indbygget. - Samarbejde og projektstyring.
  • Tiltænkt som kodedelings-site.
  • Et socialt medie.
  • Er meget udbredt.
  • Portfolio for kode.
  • Kan bruges via kommandolinje eller desktopprogram.
  • Store 'Open Source' projekter hostes her.
  • Et sted at hente store og små stykker kode til egne projekter.
  • Mulighed for private repositories. Er dog begrænset, hvis man ikke betaler.

Oprettelse af repository i GitHub

Først oprettes der en mappe der er dedikeret til kode der deles i GitHub.

Derefter kan man oprette et repository inde i desktop eller browseren.

Når man har overført sit projekt til GitHub mappen, kan man se den i desktop appen.

Bliver der lavet en ændring, holder GitHub styr på hvad der er blevet ændret.

Man kan også lave pull requests for at få noget ændret i masteren. Hvor ejeren af respositoriet kan vælge om ændringen skal blive commited til projektet.
